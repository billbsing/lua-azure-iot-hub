Module luaazureiothub

Azure Iot Hub Device SDK for Lua

This library provides the lua interface to the Microsoft Azure IotHub SDK.

See the Microsoft Azure IoT Device SDK for C documentation which this library uses to access the Azure Iot Hub.

Functions

connect (connectString[, protocol=AMQP[, processRead=nil[, processSent=nil]]])
Connect to the Azure IotHub, if successfull returns an IotHub object.

Parameters:

  • connectString string String to connect to the Azure IotHub
  • protocol string Name of the protocol (case insensitive), can be 'AMQP', 'MQTT' or 'HTTP' (default AMQP)
  • processRead function Function to process read messages, see the callback function processRead. (default nil)
  • processSent function Function to process reply after sending a message, see the callback function processSent. (default nil)

Returns:

  1. iotHub object table if successfully connected to the IotHub.
  2. false, errorMessage False and an error message if failed to connect

Usage:

    local luaazureiothub = require 'luaazureiothub'

local processRead = function(message)
  print("received a message:")
  print(message)
  return luaazureiothub.messageReceive.ACCEPTED
end

local processSendConfirmation = function(status, message)
  -- called when the message has been sent or has an error
  if status == luaazureiothub.messageSend.OK then
    print('RX message ack')
  else
    print('RX message error:' .. status)
  end
  print(message)
end

local connectionString = 'HostName=hostname.azure-devices.net;DeviceId=deviceId;SharedAccessKey=????'
local iothub, errorMessage = luaazureiothub.connect(connectionString, 'amqp', processRead, processSendConfirmation)

message
Message table used for sending or receiving via the Azure IotHub.

On sending a message, you need to only fill in the text field before passing this table to the sendMessage function.

On receving a message the text, contentType and length will have values, other fields will be set if they are present in the message.

Fields:

  • text string Mesasge text, this can be a text string or binary data with embedded zeros.
  • length number,nil If the length is provided then the text field will be encoded as binary data, with that length
  • contentType number,nil

    The message text can be overridden by using this field. It can be one of the following:

    nil                   Use the length field too decide which encoding too use string or byte.
contentType.BYTE      Always encode using byte encoding.
contentType.STRING    Always encode using string encoding.
  • id string,nil Message id, if set to nil, then the sendMessage function will automatically assign a random uuid
  • correlationId string,nil You can read/write the correlationId.
  • property table,nil Set of name="Value" pairs as property values to send with the message.

Usage:

    -- basic text message
local message = {
  text = 'Basic text message'
}
-- sends out the message
iothub:sendMessage(message)
-- also sends out the same message above
iothub:sendMessage('Basic text message')

-- basic data message with a fixed length of 12
local message = {
  text = 'Basic data message',
  length = 12
}

-- forced text message to always send out 12 character string
local message = {
  text = 'Basic data message',
  length = 12,
  contentType = luaazureiothub.contentType.STRING,
}

-- string message with a property value of 'messageType', and a special message id
local message = {
  text = 'Basic text message',
  property = {
     messageType='event',
  },
  id = 'My test id',
}

Callback Functions

These functions are called from the library to send back data and return status of sent messages from the Iot Hub.
processRead (message)
Callback function to read messages sent from the IotHub. You need to create a function with these parameters and pass the function as a parameter to the connect function.

Parameters:

  • message message The message table sent from the IotHub.

Returns:

    integer You can return values from the static table messageReceive, to accept or reject the message. If you do not return a valid integer then the library will return messageReceive.ACCEPTED as default.
processSent (status, message)
Callback function to accept a sent message. You need to create a function with these parameters and pass the function as a parameter to the connect function.

Parameters:

  • status integer The status of the sent message, see the static table messageSend for the possible values.
  • message message A copy of the message that has been sent. This message has been re-encoded from the C library so it will not have any extra fields added when used in the sendMessage function.

iotHub:loop ([timeoutSeconds=1])
Loop around the event queue to process sending and receiving of messages.

You will need to call this if you whish to receive any messages and or if you have sent a message using sendMessage setting the timoutSeconds to 0.

Parameters:

  • timeoutSeconds integer Number of seconds to loop around and process the message queues. If you set this value to <=0 then the loop will process only one cycle and return. (default 1)

Usage:

    -- send out 10 async messages
for counter = 1, 10 do
  -- send message but do not wait for reply
  iotHub:sendMessage('Test message ' .. counter, 0)
  -- process the messages sent out and maybe also read some in
  iotHub:loop()
end
iotHub:sendMessage (message[, timeoutSeconds=5])
Sends a message to the Azure IotHub

Parameters:

  • message table,string Mesasge to send, this field can be a string or a message table. If you use a string then message sent will be a simple message with string encoding.
  • timeoutSeconds number Number of seconds to wait for the Ack reply to be recieved from the IotHub

    if the timeoutSeconds == 0, then this function will return as soon as the message has been sent. It is up to the calling code to then call the loop function to wait for the message ack to be sent back from the IotHub. (default 5)

Returns:

  1. boolean,integer True if successfully sent the message, and the result code returned from the send confirmation. see the static messageSend table of possible values.
  2. boolean,string,integer False with the error message, and extra error code returned from the call to send message. See the static values in the table clientResult for the error codes returned.

Static values

These tables contain static values that are returned or set by the Azure IotHub SDK.
clientResult
Static values to define the returned error code from the call to sendMessage function.

Fields:

  • OK integer returns the IOTHUBCLIENTOK value
  • INVALID_ARG integer returns the IOTHUBCLIENTINVALID_ARG value
  • ERROR integer returns the IOTHUBCLIENTERROR value
  • INVALID_SIZE integer returns the IOTHUBCLIENTINVALID_SIZE value
  • INDEFINITE_TIME integer returns the IOTHUBCLIENTINDEFINITE_TIME value
contentType
Static values to define the message content type in the message table.

Fields:

  • STRING integer returns the IOTHUBMESSAGE_STRING value
  • BYTE integer returns the IOTHUBMESSAGE_BYTEARRAY value
messageReceive
Static values to pass back from the message recieve function

Fields:

  • ACCEPTED integer returns the IOTHUBMESSAGE_ACCEPTED value
  • REJECTED integer returns the IOTHUBMESSAGE_REJECTED value
  • ABANDONED integer returns the IOTHUBMESSAGE_ABANDONED value
messageSend
Static values that are sent from the IotHub after a message has been sent

Fields:

  • OK integer returns the IOTHUB_CLIENT_CONFIRMATION_OK value
  • DESTROY integer returns the IOTHUB_CLIENT_CONFIRMATION_BECAUSE_DESTROY value
  • TIMEOUT integer returns the IOTHUB_CLIENT_CONFIRMATION_MESSAGE_TIMEOUT value
  • ERROR integer returns the IOTHUB_CLIENT_CONFIRMATION_ERROR value
sendStatus
Static values to define the send status returned by the getSendStatus function.

Fields:

  • IDLE integer returns the IOTHUB_CLIENT_SEND_STATUS_IDLE value
  • BUSY integer returns the IOTHUB_CLIENT_SEND_STATUS_BUSY value
